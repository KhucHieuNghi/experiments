Tests

Tests are YAML files that describe user journeys in natural language. ETUS interprets each step and executes it against the browser.

Test file format

# tests/login.yaml
test-id: test_login_success
name: Successful user login
meta:
  tags: [auth, smoke]
  priority: high

use:
  baseUrl: http://localhost:3000
  browser: chromium

steps:
  - Navigate to the login page
  - Enter "user@example.com" into the email field
  - Enter "{{secret:TEST_PASSWORD}}" into the password field
  - Click the "Sign in" button
  - Verify the dashboard page loads with a welcome message

Required fields

FieldTypeDescription
test-idstringUnique identifier for the test. Use the ID generation helpers.
namestringHuman-readable test name.
stepsstring[]Ordered list of natural language instructions.

Optional fields

FieldTypeDescription
metaobjectMetadata: tags, priority, description.
useobjectPer-test config overrides (browser, baseUrl, timeout, etc.)
variablesobjectTest-scoped variable definitions.

Variables

Use double curly braces to reference variables in steps:

  • {{variableName}} — resolved from config, suite, or test variables
  • {{secret:NAME}} — resolved from environment; redacted in all output and artifacts
# Using variables in steps
variables:
  username: testuser@example.com

steps:
  - Navigate to {{baseUrl}}/login
  - Enter "{{username}}" into the email field
  - Enter "{{secret:USER_PASSWORD}}" into the password field
  - Click "Sign in"
Tip
Variables from hooks (output via /tmp/etus-agent.env) are also available in steps. See Hooks for details.

Per-test overrides

The use block lets you override any global config setting for a single test:

use:
  browser: firefox
  headless: false
  viewport: { width: 1920, height: 1080 }
  timeout: 60000
  video: true

Running a single test

npx etus-agent run tests/login.yaml

See CLI Commands for additional run options like --no-cache and --log-level.
