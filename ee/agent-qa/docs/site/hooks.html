Hooks

Hooks are sandboxed scripts that run before or after tests. Use them for database seeding, mock servers, cleanup, and environment setup.

Hook definition

Hooks are defined in a hooks.yaml file:

# hooks/hooks.yaml
hooks:
  - hook-id: hook_seed_database
    name: Seed test database
    runtime: node
    script: scripts/seed-db.mjs
    timeout: 30000
    env:
      DATABASE_URL: "{{secret:DATABASE_URL}}"

  - hook-id: hook_cleanup_data
    name: Clean up test data
    runtime: bash
    script: scripts/cleanup.sh
    timeout: 15000

Docker sandbox

Hooks run inside a Docker container with strict resource limits:

ConstraintDefault
Memory limit512 MB
CPU limit1 CPU
FilesystemRead-only (except /tmp)
NetworkDisabled by default
TimeoutDefined per hook
Info
Enable network access for a hook by adding network: true to its definition. Only enable when the hook needs to reach external services.

Variable output

Hooks can export variables to subsequent tests by writing key-value pairs to /tmp/etus-agent.env:

// scripts/seed-db.mjs
import { writeFileSync } from 'node:fs'

const userId = await seedTestUser()
const token = await generateAuthToken(userId)

writeFileSync('/tmp/etus-agent.env', [
  `SEEDED_USER_ID=${userId}`,
  `AUTH_TOKEN=${token}`,
].join('\n'))

These variables become available in test steps as {{SEEDED_USER_ID}} and {{AUTH_TOKEN}}.

Hook fields reference

FieldTypeRequiredDescription
hook-idstringYesUnique identifier. Use ID generation helpers.
namestringYesHuman-readable name.
runtimestringYesOne of: node, bun, python, bash.
scriptstringYesPath to the script file relative to hooks directory.
timeoutnumberNoMax execution time in ms (default: 30000).
envobjectNoEnvironment variables passed to the container.
networkbooleanNoEnable network access (default: false).

Using hooks in suites

Reference hooks by ID in a suite definition:

# suites/regression.yaml
hooks:
  setup:
    - hook_seed_database
  teardown:
    - hook_cleanup_data

See Suites for the complete suite format.
