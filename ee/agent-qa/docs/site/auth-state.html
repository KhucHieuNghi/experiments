Auth State

Capture authenticated browser sessions once and reuse them across tests. Keep credentials out of test artifacts and logs.

How it works

Capture a login session

Use etus-agent auth login to interactively log into your application. ETUS captures the browser state (cookies, local storage, session storage) and saves it under a logical name.

Store by name

The auth state is stored at ~/.etus-agent/auth.json with a human-readable name (e.g., "admin", "viewer", "test-user").

Reuse in tests

Tests reference the named auth state. ETUS restores the session before executing steps, so tests start already logged in.

Pass to hooks

Auth state names are available to hooks so setup scripts can make authenticated API calls.

Capturing auth state

# Interactive login capture
npx etus-agent auth login --name admin --url http://localhost:3000/login

# Test that the stored state is still valid
npx etus-agent auth test --name admin

# List stored auth states
npx etus-agent auth status

Using auth in tests

# tests/admin-dashboard.yaml
test-id: test_admin_dashboard
name: Admin can view analytics

use:
  auth: admin              # Use the "admin" auth state
  baseUrl: http://localhost:3000

steps:
  - Navigate to the analytics dashboard
  - Verify the page shows revenue metrics
  - Verify the "Export CSV" button is visible

Using auth in suites

# suites/admin-flows.yaml
suite-id: suite_admin_flows
name: Admin user flows

use:
  auth: admin

tests:
  - tests/admin-dashboard.yaml
  - tests/admin-users.yaml
  - tests/admin-settings.yaml

Credential safety

ETUS keeps credentials safe through multiple layers:

  • No plaintext passwords in test files—only logical auth state names
  • Redaction in all output, logs, and artifacts
  • Local storage only—auth state lives at ~/.etus-agent/auth.json, never in the project directory
  • Secret variables—use {{secret:NAME}} for any sensitive values; always redacted
Important
The ~/.etus-agent/auth.json file contains session tokens. Never commit it to version control. It is stored in the user home directory by design.

CLI commands

CommandDescription
etus-agent auth loginCapture a new auth state interactively
etus-agent auth testValidate a stored auth state is still valid
etus-agent auth statusList all stored auth states
etus-agent auth setManually set auth credentials
etus-agent auth logoutRemove a stored auth state
