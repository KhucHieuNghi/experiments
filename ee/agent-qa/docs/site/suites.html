Suites

Suites group related tests together with shared configuration, hooks, and execution order.

Suite file format

# suites/smoke.yaml
suite-id: suite_smoke_critical
name: Critical smoke tests

use:
  browser: chromium
  baseUrl: http://localhost:3000
  timeout: 45000

hooks:
  setup:
    - hook_seed_database
  teardown:
    - hook_cleanup_data

tests:
  - tests/login.yaml
  - tests/dashboard.yaml
  - tests/checkout.yaml

Required fields

FieldTypeDescription
suite-idstringUnique identifier. Use the ID generation helpers.
namestringHuman-readable suite name.
testsstring[]Ordered list of test file paths relative to project root.

Optional fields

FieldTypeDescription
useobjectSuite-level config overrides applied to all tests in the suite.
hooksobjectSetup and teardown hook references.
metaobjectMetadata: tags, description, owner.
variablesobjectSuite-scoped variables available to all tests.

Suite-level overrides

The use block applies to every test in the suite. Individual tests can override these settings further with their own use block.

use:
  browser: firefox
  headless: true
  video: true
  baseUrl: https://staging.example.com

Setup and teardown hooks

Hooks run in order before the first test (setup) and after the last test (teardown). Reference hooks by their ID:

hooks:
  setup:
    - hook_seed_database
    - hook_start_mock_server
  teardown:
    - hook_cleanup_data

See Hooks for details on writing hook scripts.

Running a suite

npx etus-agent run suites/smoke.yaml

ETUS executes setup hooks, then each test in order, then teardown hooks. Results are reported to the terminal and dashboard.

Tip
Suite variables propagate to all tests. Use this for shared credentials, base URLs, or environment identifiers without repeating them in each test file.
