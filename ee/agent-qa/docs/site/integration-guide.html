Integration Option A: MCP API (Recommended)

The MCP (Model Context Protocol) server exposes 33 tools over HTTP or stdio. This is the richest integration surface — supports test authoring, execution, result inspection, memory, and config management.

Connection

TransportEndpointUse Case
HTTP (Streamable)http://127.0.0.1:3471/mcpRemote clients, CI, partner APIs
stdioetus-agent mcpIDE plugins, local agents

Protocol

  • JSON-RPC 2.0 over Server-Sent Events (SSE)
  • Protocol version: 2024-11-05
  • Headers: Content-Type: application/json, Accept: application/json, text/event-stream

Quick Start

# Initialize MCP session
curl -N -X POST http://127.0.0.1:3471/mcp \
  -H "Content-Type: application/json" \
  -H "Accept: application/json, text/event-stream" \
  -d '{
    "jsonrpc": "2.0",
    "id": 1,
    "method": "initialize",
    "params": {
      "protocolVersion": "2024-11-05",
      "capabilities": {},
      "clientInfo": {"name": "partner-system", "version": "1.0"}
    }
  }'

# Response (SSE):
# event: message
# data: {"result":{"protocolVersion":"2024-11-05",
#   "capabilities":{"tools":{"listChanged":true}},
#   "serverInfo":{"name":"etus-agent","version":"0.1.21"}},...}

List Available Tools

{"jsonrpc":"2.0","id":2,"method":"tools/list","params":{}}

Call a Tool (Example: Run a Test)

{"jsonrpc":"2.0","id":3,"method":"tools/call","params":{
  "name": "etus_agent_enqueue_test_run",
  "arguments": {
    "testId": "t_alpha-bravo-charlie-delta-echo-foxtrot-golf-hotel-india-juliet",
    "target": "my-app"
  }
}}

Integration Option C: CLI (Subprocess)

For simple integrations, spawn ETUS as a subprocess. Best for CI/CD pipelines and shell-based automation.

Installation

npm install -D etus-agent

Key Commands

# Run a test (exits 0 on pass, 1 on fail)
npx etus-agent run tests/login.yaml

# Run a suite
npx etus-agent run suites/smoke.suite.yaml

# Run all discovered tests
npx etus-agent run

# Run with options
npx etus-agent run --no-cache --headless tests/checkout.yaml

# Start dashboard (background)
npx etus-agent dashboard &

# Check environment
npx etus-agent doctor

# Auth management
npx etus-agent auth set --config default --type api-key
npx etus-agent auth test --config default

CI/CD Integration (GitHub Actions)

name: E2E Tests
on: [push]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '24' }
      - run: npm ci
      - run: npx etus-agent install-browsers --chromium
      - run: npx etus-agent auth set --config default --type api-key "$LLM_KEY"
        env: { LLM_KEY: "${{ secrets.LLM_API_KEY }}" }
      - run: npx etus-agent run
      - uses: actions/upload-artifact@v4
        if: always()
        with: { name: test-results, path: .etus-agent/artifacts/ }

0All tests passed
1One or more tests failed
2Configuration error (missing config, invalid schema)

Feature: Test Execution Engine

The core agent loop processes each test step through an Observe → Plan → Execute → Verify cycle with multi-action support.

How It Works

Observe

The agent captures the current screen state: DOM tree, accessibility tree, URL, viewport metadata, and a compressed screenshot.

Plan

The LLM receives the observation + step instruction and returns an action plan (click, fill, navigate, scroll, assert, etc.) with confidence score and reasoning.

Execute

The platform adapter (Playwright) executes the planned action on the browser.

Verify

If the step isn't complete, the loop repeats. The planner signals stepComplete: true when the goal is achieved.

Self-Healing

When an action fails, ETUS does not immediately fail the test. Instead:

  • The failure is added to the sub-action history
  • The planner re-observes the screen and tries an alternative approach
  • Up to healing.maxAttempts (default: 3) consecutive failures before the step fails

Test YAML Format

test-id: t_quad-adar-micro-magic-cross-cue-open-agog-rang-cours
name: User can complete checkout
meta:
  retries: 1
steps:
  - Navigate to the product catalog
  - Add the first item to cart
  - Open the shopping cart
  - Click "Proceed to Checkout"
  - Fill in shipping address with valid data
  - Select standard shipping
  - Verify order summary displays correct total
  - Click "Place Order"
  - Verify confirmation page shows order number

Feature: Memory System

ETUS accumulates behavioral observations from test runs and injects them as context into future runs. This makes the agent progressively better at testing your specific product.

Memory Tiers

TierScopeExample
ProductApplies to all tests for a target"Login button uses aria-label 'Sign In'"
SuiteApplies to tests within a suite"After step 3, a loading spinner appears for 2s"
TestApplies to one specific test"The confirm dialog requires scrolling to see the button"

Trust Scores

Each observation has a trust score (0.0–1.0) that increases through repeated confirmation and decreases when contradicted. Low-trust observations are excluded from future runs.

Integration Value

  • Tests become more reliable over time without manual maintenance
  • The agent adapts to UI changes automatically
  • Memory is file-backed (version-controllable, reviewable)
  • Located at etus-agent-memory/ — can be shared across environments

Feature: Smart Caching

ETUS caches validated action plans at the sub-action level, dramatically reducing LLM costs and execution time on subsequent runs.

Performance Impact

5x Faster

Execution time: 42s → 8s on cached runs

3x Fewer Tokens

LLM planner calls skipped for cached actions

$0 per Cache Hit

Cached actions bypass the LLM entirely

How It Works

  • Cache key = SHA-256 of (config + suite + test + step + platform + step index)
  • Sub-actions cached individually: step with 5 actions = 5 cache entries
  • Cache miss at index N invalidates all entries from N onward (prefix invalidation)
  • TTL-based expiry (default: 7 days)
  • The agent still observes the live page — cache only skips planning, not verification

Configuration

services:
  cache:
    dir: .etus-agent/cache
    ttl: 7d    # or 24h, 1h, etc.

# Override per-run:
ETUS_AGENT_CACHE_DIR=/custom/cache etus-agent run
etus-agent run --no-cache   # bypass cache

Feature: Hook System

Hooks run before/after tests or suites in isolated Docker containers. Use them for data setup, API calls, auth preparation, and cleanup.

Sandbox Constraints

ConstraintValue
Memory512 MB
CPU1 core
PID limit256
FilesystemRead-only (except /tmp)
NetworkDisabled by default (enable with network: true)
RuntimesNode 24, Bun 1.x, Python 3.12, Bash (Alpine)

Variable Output

Hooks pass data back by writing to /tmp/etus-agent.env:

// setup-hook.mjs (inside container)
import { writeFileSync } from 'fs'

const token = await fetchAuthToken()
writeFileSync('/tmp/etus-agent.env', `AUTH_TOKEN=${token}\n`)

Subsequent hooks and test steps can access {{AUTH_TOKEN}}.

Feature: Accessibility Auditing

ETUS integrates axe-core for automated WCAG accessibility checks during test execution.

Configuration

services:
  accessibility:
    enabled: true
    standard: wcag2aa      # or wcag2aaa
    runAfter: every-step   # or end-of-test
    failOnViolation: false # true = fail test on a11y issues

Integration Value

  • Accessibility violations reported per step in run results
  • Available via MCP tools and Dashboard API
  • Non-blocking mode: log violations without failing tests
  • Blocking mode: fail tests on WCAG violations

registry:
  llms:

      provider: openai-compatible

Configuration Schema

Full configuration file: etus-agent.config.yaml

workspace:
  testMatch: ['tests/**/*.yaml']      # Glob patterns for test discovery

  hooksFile: hooks.yaml               # Hook registry
  agentRules: ./agent-rules.md        # Custom LLM planner instructions
  envFile: .env                       # Non-secret variables
  secretsFile: .env.secrets.local     # Secret values (redacted in output)

services:
  dashboard:
    port: 3100                        # Dashboard UI + API port
    artifactsDir: .etus-agent/artifacts
  mcp:
    enabled: true
    transport: http                   # http or stdio
    host: 127.0.0.1                   # loopback only by default
    port: 3471
    path: /mcp
  cache:
    dir: .etus-agent/cache
    ttl: 7d                           # Cache entry lifetime
  authState:
    dir: .etus-agent/auth-states
  accessibility:
    enabled: true
    standard: wcag2aa                 # wcag2aa or wcag2aaa
    runAfter: every-step
    failOnViolation: false
  recording:
    enabled: true                     # Playwright video recording
  memory:
    enabled: true
    provider: local
    dir: etus-agent-memory
  logging:
    level: warn                       # debug | info | warn | error

registry:
  llms:
    - name: default
      provider: openai-compatible
      model: anthropic/claude-sonnet-4
      baseURL: https://openrouter.ai/api/v1
      screenshotSize: 50kb
      effectiveResolution: 500
  targets:
    my-app:
      platform: web
      url: https://app.example.com

use:
  browser:
    name: chromium                    # chromium | firefox | webkit
    headless: true
    viewport: { width: 1280, height: 720 }
  timeout:
    step: 5m
    test: 30m
    navigation: 1m
  healing:
    maxAttempts: 3
  planner:
    maxSubActions: 10
    previousStepCount: 5
  logCapture:
    console: true
    network: true
