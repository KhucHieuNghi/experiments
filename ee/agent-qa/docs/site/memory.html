Memory

Memory is ETUS's persistent observation layer. It stores what the agent learns about your product across runs and feeds that context back into future test planning.

What is memory?

Memory is a file-backed collection of observations the agent makes during test execution. These observations capture knowledge about your application's behavior, structure, and patterns. Over time, memory helps the agent make better decisions, recover from failures faster, and produce more reliable tests.

Memory tiers

Observations are organized into three tiers based on scope:

TierDirectoryScope
Productetus-agent-memory/products/Cross-cutting knowledge about the entire application
Suiteetus-agent-memory/suites/Patterns specific to a test suite
Testetus-agent-memory/tests/Observations tied to a specific test

How it works

Run a test

ETUS executes the test steps and observes the application's behavior, DOM structure, timing, and outcomes.

Generate observations

The agent produces candidate observations about what it learned during the run.

Curate

The memory curator evaluates each observation and decides to add, update, deprecate, or do nothing.

Persist

Accepted observations are written to the appropriate tier directory as YAML files.

Feed back

On the next run, relevant memory is loaded and provided as context during test planning and execution.

The curator

The memory curator is the gatekeeper. For each candidate observation, it decides:

  • Add — new, valuable knowledge not yet captured
  • Update — refines or corrects an existing observation
  • Deprecate — marks outdated observations that no longer apply
  • Do nothing — the observation is redundant or low-value

Trust scores

Each observation carries a trust score (0.0–1.0) that reflects confidence. Scores increase when observations are confirmed across multiple runs and decrease when contradicted. Low-trust observations are weighted less during planning.

Memory vs cache vs config

ConceptWhat it storesLifetimePurpose
MemoryBehavioral observationsPersistent, curatedMake the agent smarter over time
CacheAction plans (sub-step sequences)TTL-based, invalidated on changeSpeed up repeat runs, reduce tokens
ConfigSettings and overridesStatic until changed by userControl agent behavior

Configuration

# etus-agent.config.yaml
services:
  memory:
    dir: etus-agent-memory

Use npx etus-agent clean-memory to remove deprecated observations. See CLI Commands.

Tip
Commit your etus-agent-memory/ directory to version control. Memory is part of your test infrastructure and improves team-wide test quality.
