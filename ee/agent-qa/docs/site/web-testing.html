Web Testing

ETUS uses Playwright under the hood to drive real browsers. It supports Chromium, Firefox, and WebKit with intelligent DOM extraction, smart waits, and built-in accessibility validation.

Supported browsers

BrowserConfig valueEngine
ChromiumchromiumChromium (same as Chrome/Edge)
FirefoxfirefoxGecko
WebKitwebkitWebKit (same as Safari)

Install all browsers with:

npx etus-agent install-browsers --all

DOM extraction

ETUS extracts a structured representation of the page DOM before each action. This extraction:

  • Traverses shadow DOM boundaries
  • Captures ARIA roles, labels, and states
  • Identifies interactive elements (buttons, links, inputs, selects)
  • Records element visibility and position
  • Handles dynamic content and lazy-loaded elements

Smart waits

Before acting, ETUS waits for page stability using multiple signals:

  • DOM ready — document has finished loading
  • Network idle — no pending network requests for 500ms
  • Animations complete — CSS transitions and animations have settled
  • Element stable — target element is visible, attached, and not moving

This eliminates the need for manual sleep/wait statements in tests.

Action validation

Every browser action is validated before and after execution:

  • Pre-action: element exists, is visible, is enabled, and is not obscured
  • Post-action: expected state change occurred (navigation, DOM mutation, etc.)
  • Retry: if validation fails, the agent re-evaluates and retries with alternative strategies

Accessibility checks

ETUS runs accessibility validation using axe-core against WCAG 2.0 AA standards. This includes:

  • Color contrast ratio verification
  • ARIA attribute correctness
  • Keyboard navigability
  • Form label associations
  • Image alt text presence

Accessibility violations are reported alongside test results and captured in memory for future reference.

Video recording

Enable video recording to capture the full browser session:

use:
  video: true

Videos are saved to the run artifacts directory and linked from the dashboard.

Console and network logs

ETUS captures browser console output and network requests during execution:

  • Console errors and warnings are flagged in results
  • Failed network requests (4xx, 5xx) are highlighted
  • Logs are available in the dashboard run detail view

Configuration

# etus-agent.config.yaml
use:
  browser: chromium          # chromium | firefox | webkit
  headless: true             # Run without visible browser window
  viewport:
    width: 1280
    height: 720
  video: false               # Record browser session
  timeout: 30000             # Action timeout in ms
Tip
Use ETUS_AGENT_HEADLESS=false during local development to watch the agent interact with the page in real time.
