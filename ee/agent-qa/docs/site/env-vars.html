Environment Variables

All ETUS environment variables use the ETUS_AGENT_ prefix. They override corresponding config file values.

Variable reference

VariableConfig equivalentDescription
ETUS_AGENT_DASHBOARD_PORT services.dashboard.port Port for the local dashboard server (default: 4100)
ETUS_AGENT_MCP_PORT services.mcp.port Port for the MCP server (default: 4200)
ETUS_AGENT_CACHE_DIR services.cache.dir Directory for action plan cache (default: .etus-agent/cache)
ETUS_AGENT_CACHE_TTL services.cache.ttl Cache time-to-live in seconds (default: 86400)
ETUS_AGENT_LOG_LEVEL — Log level: silent, error, warn, info, debug
ETUS_AGENT_HEADLESS use.headless Run browsers in headless mode (true/false)
ETUS_AGENT_CLI_BIN — Path to the etus-agent binary. Used by the dashboard to spawn child processes.

How overrides work

Environment variables sit between CLI flags and the local override file in the resolution order:

  1. CLI flags (highest priority)
  2. Environment variables
  3. Per-test use overrides
  4. Suite-level use overrides
  5. etus-agent.local.yaml
  6. etus-agent.config.yaml (lowest priority)

CI example

In CI environments, set variables to control behavior without modifying config files:

# GitHub Actions example
env:
  ETUS_AGENT_HEADLESS: "true"
  ETUS_AGENT_LOG_LEVEL: "info"
  ETUS_AGENT_CACHE_TTL: "3600"
  OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}
Info
LLM API keys (e.g., OPENAI_API_KEY) are referenced in the config file via ${OPENAI_API_KEY} syntax. They do not use the ETUS_AGENT_ prefix because they belong to the LLM provider, not ETUS.

Dashboard child process

The ETUS_AGENT_CLI_BIN variable tells the dashboard where to find the CLI binary when spawning test runs. This is useful in monorepo setups or when the CLI is installed globally:

export ETUS_AGENT_CLI_BIN=./node_modules/.bin/etus-agent
