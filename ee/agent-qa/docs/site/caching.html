Caching

ETUS caches action plans to dramatically speed up repeat test runs. Cached runs are up to 5x faster and use 3x fewer LLM tokens.

What gets cached

When ETUS interprets a test step, it generates a sub-action plan—the sequence of browser actions to perform. These plans are cached so that identical steps don't require another LLM call on subsequent runs.

Cache key

Each cache entry is keyed by a SHA-256 hash of:

  • Resolved configuration
  • Suite context (if running within a suite)
  • Test definition
  • Step text
  • Platform (browser + viewport)
  • Step index within the test

Changing any of these inputs produces a new key, causing a cache miss.

Hit, miss, and invalidation

ScenarioBehavior
Cache hitCached plan is loaded and executed directly. No LLM call.
Cache missLLM generates a new plan. Result is cached for future runs.
TTL expiredEntry is treated as a miss and regenerated.
Step text changedNew key is generated; old entry is orphaned.

Prefix invalidation

You can invalidate all cache entries for a specific test or suite by purging with a prefix:

# Purge all entries for a specific test
npx etus-agent cache purge --prefix test_login_success

# Purge all entries for a suite
npx etus-agent cache purge --prefix suite_smoke_critical

# Purge everything
npx etus-agent cache purge --all

Configuration

# etus-agent.config.yaml
services:
  cache:
    dir: .etus-agent/cache
    ttl: 86400             # Time-to-live in seconds (default: 24 hours)

Environment variable overrides:

  • ETUS_AGENT_CACHE_DIR — override the cache directory path
  • ETUS_AGENT_CACHE_TTL — override the TTL value in seconds

Disabling the cache

Use the --no-cache flag to skip the cache for a single run:

npx etus-agent run tests/login.yaml --no-cache

This forces fresh LLM calls for every step but still writes new entries to the cache.

Performance impact

5x faster

Cached runs skip LLM round-trips entirely, executing plans in milliseconds.

3x fewer tokens

Each cache hit eliminates the prompt and completion tokens for that step.

Deterministic

Cached plans produce consistent behavior across runs, reducing flakiness.

Info
The cache directory (.etus-agent/cache) should be in your .gitignore. It is machine-specific and regenerates automatically.
