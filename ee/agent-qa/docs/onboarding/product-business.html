Product and Business Onboarding

How ETUS turns natural-language YAML test intent into an executed QA result that humans and coding agents can triage.

Overview

Readiness: PASS

The primary business value is local, reviewable, agent-assisted QA execution for web and mobile products. The system reduces selector maintenance, records detailed execution evidence, and improves repeated runs through cache and memory.

Journey

The journey starts when a developer or agent authors or selects a YAML test or suite. The CLI, dashboard, or MCP validates and runs it. The core runtime observes UI, plans typed actions, executes through platform adapters, verifies completion, writes artifacts, and exposes results for review.

Rules

  • Tests require canonical IDs, target, name, and at least one step.
  • Global platform selection is rejected; target-level platform resolution owns this boundary.
  • Cache and memory can be disabled per run.
  • Mobile jobs serialize per platform in the dashboard queue.
  • MCP tools delegate run and authoring operations through local dashboard APIs.
