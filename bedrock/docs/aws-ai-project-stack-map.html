AWS AI ecosystem project map

Chọn đúng stack AWS AI cho dự án mới

Trang này gom lại Bedrock, AgentCore, SageMaker, Amazon Q, data/RAG/search, runtime và security/ops thành một bản đồ triển khai. Mục tiêu là giúp một human nhìn vào và biết request đi qua đâu, stack nào nên dùng, và cần chuẩn bị tài liệu gì trước khi build.

Muốn dùng ngay Ưu tiên Amazon Q hoặc Applied AI APIs nếu bài toán khớp sản phẩm AWS có sẵn.
Muốn build genAI app Ưu tiên Bedrock khi cần model, prompt, RAG, guardrails, flows.
Muốn agent production Ưu tiên AgentCore khi agent cần tools, memory, identity, observability, evaluation và policy.

1. Cách đọc tài liệu này

Đọc theo thứ tự này nếu anh đang chuẩn bị một dự án mới. C4 giúp nhìn architecture, stack map giúp chọn service, request journey giúp hiểu runtime flow.

Step 1 Nhìn overview architecture Xác định request đi qua edge, security, orchestration, AI layer, data/tools và ops loop.
Step 2 Drill-down bằng C4 Xác định system context, containers, components và implementation entry points.
Step 3 Chọn stack AWS Amazon Q, Bedrock, AgentCore, SageMaker, AI APIs, data/search, runtime, ops.
Step 4 Map journey và roadmap Chọn kiểu request rồi triển khai theo phase: prototype, production, evaluation, pilot.

2. Architecture overview: request đi qua AWS như thế nào?

Diagram này đọc theo hướng trái sang phải. Đường liền là data plane của request; đường nét đứt là control plane/observability chạy song song để deploy, govern và cải thiện hệ thống.

Data plane Luồng request thật: user → app edge → security → orchestration → AI/data/tools → response.
AI decision layer Điểm chọn Bedrock, AgentCore, SageMaker hoặc Amazon Q theo bản chất request.
Data/tool layer Nơi agent/RAG/model lấy knowledge, gọi Lambda/API/MCP, hoặc dùng AI API có sẵn.
Control plane Deploy, observe, evaluate, govern và cải thiện hệ thống liên tục.

3. C4 drill-down cho AWS AI project

C4 giúp đi sâu từ bối cảnh hệ thống tới containers, components và implementation entry points. Phần này bổ sung cho architecture overview ở trên, không thay thế nó.

C4-L1

System Context: ai tương tác với AI project?

Level này trả lời: system AI nằm giữa người dùng, enterprise data, tools nội bộ và đội vận hành như thế nào.

Users / Employees Hỏi đáp, yêu cầu hành động, upload tài liệu, dùng assistant trong app, IDE hoặc chat.
AWS AI Project System Boundary của dự án: UI/API, orchestration, AI decision layer, data/tool layer, governance.
Enterprise systems Documents, databases, APIs, ticketing, CRM, ERP, data lake, MCP servers, Lambda tools.
Platform / Security teams IAM, network, KMS, audit, evaluation gates, incident response, cost and quota controls.
C4-L2

Container View: runtime containers trong AWS

Level này giúp chọn nơi request chạy: app edge, workflow/orchestration, AI work zone, data/tool plane và observability/governance.

Implementation Entry Points: bắt đầu build ở đâu?

Level 4 thường là code-level; với tài liệu định hướng này, nó được diễn giải thành các entry points triển khai thực tế.

AgentCore project Bắt đầu bằng AgentCore CLI, deploy Runtime/Harness, rồi thêm Memory, Gateway, Identity, Policy.
SageMaker ML endpoint Bắt đầu với dataset, experiment, training/notebook, endpoint, Model Monitor và Model Card.
Amazon Q rollout Bắt đầu với application, data connectors, ACL mapping, pilot group và feedback loop.

5. Chọn stack theo câu hỏi triển khai

Đừng bắt đầu bằng tên service. Bắt đầu bằng loại request và mức tự động hóa cần có.

Người dùng chỉ cần assistant nội bộ?

Chọn Amazon Q Business. Ưu tiên nếu cần permission-aware enterprise search nhanh.

App cần hỏi đáp trên tài liệu?

Chọn Bedrock + Knowledge Bases + Guardrails. Agent chưa cần thiết nếu workflow còn tuyến tính.

Agent phải gọi tool nhiều bước?

Chọn AgentCore. Gateway, Identity, Memory, Policy, Observability là trục production.

Cần train/custom model?

Chọn SageMaker AI. Dùng cho data science, training, endpoints, MLOps và governance.

Task là OCR/speech/translate/vision?

Chọn Applied AI APIs. Sau đó đưa kết quả vào Bedrock/AgentCore nếu cần reasoning tiếp.

6. Request journey diagram

Bốn tab dưới đây là bốn kiểu đường đi khác nhau của một request. Chúng trả lời câu hỏi: request này cần “trả lời bằng dữ liệu”, “hành động như agent”, “dự đoán bằng model ML”, hay “dùng assistant có sẵn”.

RAG answer Request cần câu trả lời dựa trên tài liệu/data. Trục chính là Bedrock + Knowledge Bases + Guardrails.
Agent action Request cần agent lập kế hoạch và gọi tools nhiều bước. Trục chính là AgentCore Runtime/Harness + Gateway + Identity.
ML prediction Request cần score/classification/ranking từ custom model. Trục chính là SageMaker endpoint và MLOps loop.
Amazon Q Request phù hợp assistant có sẵn cho employee/developer. Trục chính là Amazon Q + enterprise permission connectors.
Trace
Request id, spans, model/tool latency, errors.
Evaluate
Golden set, online sampling, quality thresholds.
Govern
IAM, guardrails, policy, audit, approval flow.
Improve
Prompt, data, tool schema, memory, model retraining.

7. Roadmap triển khai dự án mới

Đây là luồng triển khai phù hợp cho dự án mới: đi từ use case, security, data, prototype, production hardening tới pilot.

Phase 0

Scope and decision

  • Viết use-case brief.
  • Chọn Amazon Q, Bedrock, AgentCore, SageMaker hay API AI.
  • Chốt success criteria, budget, latency.
Phase 1

Security baseline

  • Account, region, IAM, KMS, VPC.
  • Tenant boundary, data residency.
  • CloudTrail, CloudWatch, quota alerts.
Phase 2

Data and tool contract

  • Source inventory, owner, ACL.
  • RAG chunking/vector/search design.
  • Tool schemas, idempotency, rate limit.
Phase 3

Prototype

  • Bedrock prompt/RAG or AgentCore CLI spike.
  • SageMaker notebook/training if ML.
  • Local tests with representative requests.
Phase 4

Production path

  • Runtime: Lambda/ECS/EKS/AgentCore/SageMaker endpoint.
  • Gateway, Identity, Memory, Policy where needed.
  • IaC with CDK, CloudFormation or Terraform.
Phase 5

Quality loop

  • Golden dataset and evaluator rubric.
  • Online sampling, traces, dashboards.
  • Regression gate before rollout.
Phase 6

Pilot

  • Small user group, feedback loop.
  • Incident, cost spike, quota runbook.
  • Rollback and escalation path.
Phase 7

Scale

  • Tool registry and ownership.
  • Memory/data retention governance.
  • Optimization, A/B testing, retraining.
